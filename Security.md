# Security Implementation
# LeadNova AI CRM

---

## 1. Authentication Security

### Google OAuth 2.0
- No passwords stored in our system
- Google handles credential verification
- OAuth tokens managed by Supabase
- Session cookies are httpOnly and secure

### Session Management
- JWT tokens stored in secure cookies (not localStorage)
- Cookie attributes: httpOnly, secure, sameSite=lax
- Session refresh handled automatically by Supabase SSR
- Logout clears all session cookies

### Middleware Protection
- Next.js middleware runs on every request
- Checks for valid Supabase session
- Unauthenticated users redirected to landing page
- Dashboard routes (/dashboard/*) are all protected

---

## 2. Database Security

### Row Level Security (RLS)
- Enabled on ALL tables (profiles, leads, emails)
- Each user can ONLY access their own data
- Policies use auth.uid() to verify ownership
- Even if API is bypassed, database rejects unauthorized queries

### Policies
```sql
-- User can only see their own leads
CREATE POLICY "Users can view own leads" 
  ON leads FOR SELECT 
  USING (auth.uid() = user_id);

-- User can only insert leads with their own user_id
CREATE POLICY "Users can insert own leads" 
  ON leads FOR INSERT 
  WITH CHECK (auth.uid() = user_id);

  SQL Injection Protection
Supabase SDK uses parameterized queries
No raw SQL concatenation in application code
All inputs go through Supabase client which handles escaping
